Explantion of the topics:

  1. Media Query Introduction:

    • Media queries used to apply CSS styles depending on a device's media type (such as print vs. screen) or other features or characteristics such as screen resolution or orientation, aspect ratio, browser viewport width or height, user preferences such as data usage.
    • Media queries helps in building responsive web designe. Responsive web design is a web design approach to make web pages render well on all screen sizes and resolutions while ensuring good usability
    • For appling media queries, we have to write @media keyword with some argument (media features) as parameter and after that inside 2nd brackets the css code will be written.
    • Basic Structure:
    • In this above example the styling mentioned inside the media query will only execute when the viewport width is less then or equal to 600px.
    • only: The only keyword is specifies that only apply the styles mentioned inside media query when the browser supports media query. Older browsers with no media query support will not display the styling mentioned inside media query.
    • The only keyword is optional. But if we use The only keyword, then we must also specify a media type.

  2. Media type

    • Media types define the broad category of device for which the media query applies: all, print, screen.
      • all: Used for all media type devices.
      • print: Used for print preview mode.
      • screen: Used for computer screens, tablets, smart-phones etc.
    • Media type is optional (assumed to be all) except when using the only logical operator.

  3. Media features

    • Media features describe the condition for media query.
    • Each media feature expression must be surrounded by parentheses.
    • Here are some commonly used media features:
      • max-width: (Mostly used), Media query will execute only when    viewport width <= value.   Eg: @media (max-width: 900px;){/*CSS*/}
      • min-width: (Mostly used), Media query will execute only when    viewport width >= value.   Eg: @media (min-width: 900px;){/*CSS*/}
      • width: Media query will execute only when    viewport width = value.   Eg: @media (width: 900px;){/*CSS*/}
      • max-height: Media query will execute only when    viewport height <= value.   Eg: @media (max-height: 500px;){/*CSS*/}
      • min-height: Media query will execute only when    viewport height >= value.   Eg: @media (min-height: 500px;){/*CSS*/}
      • height: Media query will execute only when    viewport height = value.   Eg: @media (height: 500px;){/*CSS*/}
      • orientation: It has 2 values portrait (height is greater than or equal to the width) & landscape (width is greater than the height) .   Eg: @media (orientation: portrait;){/*CSS*/}
      • etc.
    • If there is a conflict between 2 media query's media-features then the query which is written latter will execute.
    • Example: Go

  4. Logical operators:

    • The logical operators and, or, not can be used to connect multiple media features logically.
      • and: (Mostly used) Both media features together should be true for executing the query. It is also used for joining media features with media types.
        Example: @media (max-width:600px) and (min-width:400px) { CSS }.
      • or: If any 1 of media features is true then the query will execute. We can also use comma symbol(,) in the place of "or".
        Example: @media (max-width:600px) or (min-width:400px) { CSS }.
      • not: Used to negate a media query. If the media feature is false then the query will execute.
        Example: @media not(max-width:600px) { CSS }.
    • The W3C has published several levels of media queries, each extending and superseding the features of the previous level.
    • In Media Queries Level-4 >, <, = operators are introduced.
      Example: @media (width <= 600px) { CSS }

  5. Common responsive breakpoints:

    • 300px - 480px      (Mobile)
    • 481px - 768px      (Tablet)
    • 769px - 1024px    (Laptop or small desktop)
    • 1025px - up          (Large Desktop)
    • These values are for general idea purpose only. We can use different breakpoints for achieving our desired designe.

  6. Mobile first vs Desktop first designe approach:

    • Mobile first approach: Mobile-first is when we start by writing our CSS for mobile devices and then use media queries to add in styling for larger screen sizes. This is good for those web sites where most of the traffic came from mobile devices.
      Pros:
      • Faster load times on mobile devices.
      • Better user experience on mobile devices.

      Cons:
      • More difficult to create a desktop version from a mobile version.

    • Desktop first approach: Desktop-first is when we start by writing our CSS for larger screen and then use media queries to add in styling for smaller screen sizes. This is good for those web sites where most of the traffic came from desktop devices.
      Pros:
      • Easier to create a mobile version from a desktop version.
      • A more detailed and dynamic desktop experience.

      Cons:
      • Slower load times on mobile devices.
