Chapter 8: CSS Positioning



Topics we are going to learn in this page:

  1. position property
  2. Static Positioning (position: static;)
  3. Relative Positioning (position: relative;)
  4. z-index property
  5. Absolute Positioning (position: absolute;)
  6. Fixed Positioning (position: fixed;)

Explantion of the topics:

  1. position property:

    • Positioning: Positioning allows us to take elements out of normal document flow and make them behave differently.
    • We can set the positioning using these 6 css properties:
      • position
      • top
      • bottom
      • left
      • right
      • z-index
    • The position property sets how an element is positioned (static, relative, absolute, fixed, sticky) in a document.
    • The top & bottom CSS property specify the vertical shifting of a positioned element.
      (Positioned element means, element with any position value other than default value "static")
    • We should not use top & bottom together for the same element. If both are specified then top wins.
    • The left & right CSS property specify the horizontal shifting of a positioned element.
    • We should not use left & right together for the same element. If both are specified then left wins (when language direction is left to right).
    • The z-index specify which element will be on the top if overlapping happen.


  2. Static Positioning (position: static;):

    • For all HTML elements the default value of position property is static.
    • The element is positioned according to the Normal Flow of the document.
    • The top, right, bottom, left, and z-index properties have no effect.
    • Example: Go

  3. Relative Positioning (position: relative;):

    • The element is positioned according to the normal flow of the document.
    • The top, right, bottom, left, and z-index properties are used to shift the element.
    • Shifting is relative to its default position.
    • Even if the element is shifted, its initial space (with margin) in the document is maintained. (The offset does not affect the position of any other elements)
    • Example: Go

  4. z-index property:

    • The z-index property determine which element will go on the top (along Z-axis) if overlapping happen between two or more positioned elements.
      (Positioned element means, element with any position value other than "static")
    • An element with greater z-index will be on the top of an element with a lower z-index.
    • Default value 0. The value can be both positive & negative.
    • If two positioned elements overlap without a z-index specified, the element positioned last in the HTML code will be shown on top.

  5. Absolute Positioning (position: absolute;):

    • The element is removed the normal flow of the document.
    • No space is left (reserved) where the element used to be.(The offset affects the position of other elements)
    • The element is positioned relative to its closest positioned ancestor.(Positioned element means, element with any position value other than "static")
    • If there is no positioned ancestor then its position is relative to root element (top left corner of the page).
    • The top, right, bottom, left, and z-index properties are used to shift the position of the element.

  6. Fixed Positioning (position: fixed;):

    • The element is removed the normal flow of the document.
    • No space is left (reserved) where the element used to be.(The offset affects the position of other elements)
    • The element is positioned relative to the viewport window.
    • The top, right, bottom, left, and z-index properties are used to shift the position of the element.
    • When we scroll the page, the element always stay in place.
